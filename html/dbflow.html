java에서 database 자료 가져오는 방법

필요한 자료 준비

java에서 database에 접속해서 자료를 가져오기 위해서는 다음의 자료가 필요하다.
  • 접속할 서버 주소
  • 접속할 서버에서 사용하는 포트 번호
  • DBMS의 데이터베이스 이름
  • DBMS의 사용자 이름 / 비밀번호

접속할 서버 주소의 형식

jdbc:postgrespl://127.0.0.1:43432/testdb

위와 같은 형식으로 적는데 순서대로,
  • DB와 연동하는데 jdbc를 이용(java니까)
  • postgresql을 이용
  • 주소는 127.0.0.1
  • 포트번호는 43432
  • 데이터베이스 이름은 testdb
라는 걸 알려준다.

실제접속



그 밖에, invalid value for parameter "lc_messages": "en_US.utf8ˮ 해결방법



위 이미지처럼 도커 실행시 다음과 같은 에러발생시 다음과 같은 순서로 에러를 해결하면 된다.
  • powershell을 관리자권한으로 실행한다.
  • docker cp postgres:/var/lib/postgresql/data/postgresql.conf . 을 실행하여 postgresql.conf 파일을 다운로드 받는다.
  • postgresql.conf파일을 메모장으로 열어 Ctrl+F로 utf8을 검색한다.
  • 검색된 'en_US.utf8'을 'C.UTF-8'로 변경하여 저장한다음
  • 다시 powershell을 실행시킨 뒤, docker cp postgresql.conf postgres:/var/lib/postgresql/data/postgresql.conf . 해서 다시 내보낸다.
  • 내보낸 후 다시 도커를 run 하면, 이제는 testdb에 접속이 안되는 현상이 발생하게 되는데
  • locale-gen en_US.utf8 하여 다시 'C.UTF-8'을 'en_US.utf8'로 돌려 놓고 도커 재실행해야한다.
